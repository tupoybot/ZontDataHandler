# Промпт для Codex на этап 4

```text
Ты работаешь в уже существующем Android-проекте, где этапы 1, 2 и 3 уже завершены: рабочая вертикаль `phone -> watch -> complications` реализована, релиз `0.1` подготовлен, а текущие complication providers уже работают на реальных часах и эмуляторах.

Перед началом реализации обязательно прочитай:
- `docs/stage-04-plan.md`
- `docs/development-plan.md`
- при необходимости текущую реализацию этапов 1, 2 и 3
- `assets/icons/burner.svg`
- `assets/icons/coolant.svg`

Сначала согласуй свою реализацию с этими документами, затем переходи к коду.

Контекст:
- репозиторий публичный, поэтому никаких секретов в git;
- приложение не нужно публиковать в Play Market;
- этапы 1, 2 и 3 уже работают, их нельзя ломать;
- пользователь тестирует на Android emulator, Wear OS emulator, Galaxy S23+ и Galaxy Watch Ultra;
- `wear`-приложение остаётся обязательной частью решения;
- на этапе 4 главный фокус: пиктограммы, визуальная иерархия и читаемость complications;
- Samsung-specific special slot на этом этапе не является целью и отложен на этап 5.

Что уже должно остаться рабочим:
- синхронизация `phone -> watch`;
- текущие 7 providers:
  - `ZONT overview`
  - `ZONT room`
  - `ZONT burner`
  - `ZONT setpoint`
  - `ZONT coolant`
  - `ZONT setpoint + coolant`
  - `ZONT room + air setpoint`
- текущая логика placeholder/stale;
- подтверждённый маппинг реального устройства:
  - `roomTemperature` — комнатная температура;
  - `burnerModulation` — модуляция горелки;
  - `targetTemperature` для подтверждённого `z3k-state` устройства — уставка теплоносителя (`setpoint_temp`), а не температура воздуха;
  - `coolantTemperature` — фактическая температура теплоносителя;
  - `roomSetpointTemperature` — отдельная желаемая температура воздуха в помещении.

Что нужно сделать:
1. Сохрани текущую рабочую вертикаль и не ломай существующие 7 providers.
2. Обнови иконки для `burner` и `coolant`, используя SVG-исходники из `assets`.
3. Для `ZONT setpoint + coolant` используй ту же иконку, что и для `ZONT coolant`.
4. Исправь visual hierarchy в paired complications:
   - текущее значение должно быть акцентным;
   - уставка должна быть вторичной;
   - это касается как минимум `ZONT setpoint + coolant` и `ZONT room + air setpoint`.
5. Переделай `ZONT overview`, чтобы он не выглядел как случайный перенос строки:
   - не полагайся на автоматический wrap;
   - лучше сделай предсказуемый двухстрочный вид;
   - рабочая гипотеза: 2 значения в первой строке и 2 значения во второй.
6. Если visual-правки затрагивают preview data, formatter, manifest или ресурсы, обнови их согласованно.
7. После каждой успешной сборки устанавливай всё собранное, что подходит по платформе, на доступные target'ы:
   - `mobile` на Android emulator и/или телефон;
   - `wear` на Wear OS emulator и/или часы;
   - если доступны и эмуляторы, и реальные часы, используй их все для проверки.
8. Если debug/release APK конфликтуют между собой по подписи или установке, не скрывай это:
   - зафиксируй ограничение;
   - установи хотя бы тот вариант, который реально используется для ручной проверки.
9. Не трать этап 4 на Samsung special slot. Зафиксируй его как задачу этапа 5.
10. Не делай commit, tag, merge или push в `main` без явной команды пользователя.

Ограничения:
- не переделывай архитектуру заново;
- не добавляй сервер, базу данных и лишнюю инфраструктуру;
- не ломай текущие complication layouts сверх нужного visual fix;
- не используй реальные секреты в репозитории;
- не превращай этап 4 в бесконечный редизайн одного watch face;
- не уходи в Samsung-specific ветку на этом этапе.

Технические предпочтения:
- Kotlin;
- текущая архитектура проекта должна остаться узнаваемой;
- использовать существующий Wear OS complications API проекта;
- минимальные и понятные изменения без переусложнения;
- SVG из `assets` можно использовать как source of truth для новых drawable-ресурсов.

Что важно по UX:
- текущий рабочий пользовательский путь не должен стать хуже;
- paired complications должны визуально подчёркивать текущее значение, а не уставку;
- combined complication должен выглядеть аккуратно на маленьком экране и не разваливаться переносом;
- пиктограммы должны стать читаемее и спокойнее на реальных часах;
- отрицательный результат по Samsung special slot не нужно переисследовать на этапе 4.

Что проверить перед завершением:
- проект собирается;
- старая работающая вертикаль не сломана;
- данные по-прежнему доходят до часов;
- существующие 7 providers работают;
- подтверждённый маппинг метрик не регрессировал;
- `burner` и `coolant` получили новые иконки;
- `ZONT setpoint + coolant` использует иконку `coolant`;
- paired complications показывают текущее значение как акцентное;
- `ZONT overview` выглядит как аккуратный двухстрочный блок, а не как случайный перенос;
- собранные APK установлены на доступные target'ы;
- Samsung special slot явно оставлен на этап 5;
- commit/push не делался без команды пользователя.

Если какие-то детали visual hierarchy или combined layout спорные, выбирай самое простое решение, которое делает экран аккуратнее и не тащит за собой новый редизайн всей системы.
```
